# 云服务器环境配置清单

项目已在 `/root/localwiseAI` 时，在服务器上按顺序执行以下命令。

---

## 一、安装 Docker

```bash
apt update && apt install -y curl
curl -fsSL https://get.docker.com | sh
systemctl enable docker
systemctl start docker
```

验证：

```bash
docker --version
docker compose version
```

---

## 二、配置环境变量

```bash
cd /root/localwiseAI
cp app/.env.example app/.env
nano app/.env
```

在编辑器中修改：

- `DEEPSEEK_API_KEY` 改为你的真实 API 密钥（必填）
- 其他项可暂时不改

保存退出：`Ctrl+O` 回车，`Ctrl+X`。

---

## 三、为 Docker Compose 准备根目录 .env（可选）

`docker-compose.yml` 会从**项目根目录**的 `.env` 读取变量。若根目录没有 `.env`，可复制一份：

```bash
cd /root/localwiseAI
cp app/.env.example .env
nano .env
```

同样把 `DEEPSEEK_API_KEY` 改成真实密钥后保存。

---

## 四、构建并启动

```bash
cd /root/localwiseAI
docker compose up -d --build
```

首次构建可能需要 3～5 分钟，完成后会显示两个容器：`foodwise-backend`、`foodwise-frontend`。

---

## 五、检查运行状态

```bash
docker compose ps
```

应看到 `foodwise-backend` 和 `foodwise-frontend` 均为 `Up`。

---

## 六、访问

- **前端页面（必带端口）**：`http://你的服务器IP:8080`  
  例如：**http://38.95.75.203:8080**
- 后端 API：`http://你的服务器IP:8001`

注意：直接访问 `http://IP`（不带端口）会打开系统 Nginx 默认页，不是本应用。请务必使用 **:8080**。

---

## 常见问题

| 情况 | 处理 |
|------|------|
| 报错 port 8000/80 already in use | 已改为 8080（前端）、8001（后端），与系统 Nginx 不冲突。若仍冲突可再改 `docker-compose.yml` 里的端口映射。 |
| 8080/8001 打不开 | 在云控制台安全组/防火墙中放行 **8080、8001** 端口 |
| 构建报错 | 执行 `docker compose logs -f` 查看具体错误 |
| 更新代码后 | `git pull` 后执行 `docker compose up -d --build` |
| 停止服务 | `docker compose down` |

---

## 七、域名配置（可选）

已有域名 `localwise.top` 时，可用 Nginx 反向代理实现不带端口访问。

### 1. DNS 添加记录

在域名服务商后台添加：

| 类型 | 主机记录 | 值 |
|------|----------|-----|
| A | @ | 38.95.75.203 |
| A | www | 38.95.75.203 |

### 2. 配置 Nginx

```bash
cat > /etc/nginx/sites-available/localwise.top << 'EOF'
server {
    listen 80;
    server_name localwise.top www.localwise.top;

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_cache_bypass $http_upgrade;
    }

    location /chat {
        proxy_pass http://127.0.0.1:8001;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_buffering off;
        proxy_read_timeout 86400s;
        proxy_send_timeout 86400s;
    }
}
EOF
```

### 3. 启用并重启

```bash
ln -sf /etc/nginx/sites-available/localwise.top /etc/nginx/sites-enabled/
nginx -t
nginx -s reload
```

### 4. 访问

浏览器打开：**http://localwise.top**

## 八、502 / 聊天请求失败 排查

出现 **502** 或页面能打开但 **「请求失败, 请重试!」** 时，在服务器上按顺序检查。

### 1. 确认容器在运行

```bash
cd /root/localwiseAI
docker compose ps
```

两个容器都应为 `Up`。若有 `Exit`，执行：

```bash
docker compose up -d
```

### 2. 确认端口通

```bash
curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:8080
curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:8001/health
```

应分别返回 `200`。若 8080 非 200，说明前端容器异常；若 8001 非 200，说明后端未启动或崩溃。

### 3. 看后端日志（聊天失败时必看）

```bash
docker compose logs backend --tail 50
```

若出现 `DEEPSEEK_API_KEY`、`api_key` 或 401/500，说明环境变量未配置或 API Key 无效。处理方式：

```bash
cd /root/localwiseAI
nano .env
```

确认有且仅有一行（可无引号）：

```bash
DEEPSEEK_API_KEY=你的真实key
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-chat
```

保存后执行：

```bash
docker compose up -d backend
```

### 4. 确认 Nginx 代理的是 8080 / 8001

```bash
grep -A2 "proxy_pass" /etc/nginx/sites-enabled/localwise.top
```

应看到 `127.0.0.1:8080` 和 `127.0.0.1:8001`。若之前改过 Docker 端口，这里也要一致。改完后：

```bash
nginx -t
nginx -s reload
```

### 5. www 也 502 时

确认配置里包含 `www.localwise.top`：

```bash
grep server_name /etc/nginx/sites-enabled/localwise.top
```

应为 `server_name localwise.top www.localwise.top;`。若没有 `www`，用「七、域名配置」里的配置覆盖后重新 `nginx -s reload`。
